# Tietoturva

Olemme tehneet parhaamme valitaksemme esimerkkeihin ja tehtäviin vain välttämättömät ja yleisesti luotettavina pidetyt riippuvuudet. Tästä huolimatta, kuten kaikessa ohjelmistokehityksessä, aina on mahdollisuus virheisiin tai tietoturva-aukkoihin.

Myös tunnetuista ja laajasti käytetyistä kirjastoista paljastuu ajoittain haavoittuvuuksia. Tietoturva ei ole kertaluonteinen tehtävä, vaan jatkuva prosessi. Haavoittuvuuksien seuraaminen, niihin reagoiminen sekä riippuvuuksien päivittäminen ovat tärkeä osa kaikissa ohjelmistoprojekteissa.

Ole myös itse tarkkana ja asenna vain luotettavista lähteistä peräisin olevia ohjelmistoja, paketteja, laajennoksia ja muita tiedostoja. Tarvittaessa kysy neuvoa ohjaajilta ja kollegoilta.

Jos sinulla on kysyttävää, huomaat mahdollisen tietoturvaongelman oppimateriaaleissa tai havaitset jotain epäilyttävää tehtävissä tai niiden riippuvuuksissa, ota viipymättä yhteyttä opettajaasi ja projektin ylläpitäjään.

Kiitos, että autat pitämään projektin turvallisena!


# Security

We have done our best to select only necessary dependencies that are generally considered reliable for the examples and exercises. Nevertheless, as with all software development, there is always the possibility of errors or security vulnerabilities.

Even well-known and widely used libraries occasionally reveal vulnerabilities. Information security is not a one-time task, but an ongoing process. Monitoring vulnerabilities, responding to them, and updating dependencies are an essential part of all software projects.

You should also remain vigilant yourself and install software, packages, extensions, and other files only from trusted sources. If needed, ask for advice from instructors and colleagues.

If you have any questions, notice a potential security issue in the learning materials, or observe anything suspicious in the exercises or their dependencies, please contact your teacher and the project maintainer without delay.

Thank you for helping keep the project secure!
